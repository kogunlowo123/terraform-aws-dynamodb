terraform-aws-dynamodb

Architecture Document — Full-Featured AWS DynamoDB Table Module

Executive Summary

Terraform module to provision an AWS DynamoDB table with full-featured support for global tables (multi-region replication), DAX in-memory caching, auto-scaling with target tracking policies, server-side encryption (AWS or customer-managed KMS), DynamoDB Streams, Kinesis streaming, TTL, Contributor Insights, and configurable secondary indexes (GSI/LSI). Supports both PAY_PER_REQUEST and PROVISIONED billing modes.

Data Flow

1
Application sends read/write requests, optionally routed through the DAX cache cluster for microsecond-latency reads.
2
DynamoDB table processes requests using hash and optional range keys, querying via GSIs or LSIs as needed.
3
Server-side encryption protects data at rest with AWS-managed or customer-managed KMS keys.
4
DynamoDB Streams capture item-level changes (NEW_AND_OLD_IMAGES) for event-driven processing.
5
Kinesis streaming destination forwards change data to Kinesis Data Streams for real-time analytics.
6
Global table replicas synchronize writes across regions for low-latency multi-region access.
7
Auto scaling adjusts read/write capacity based on utilization targets (default 70%), preventing throttling.

Security Controls

Server-Side Encryption

Encryption at rest enabled by default (enable_encryption = true) with optional customer-managed KMS key via kms_key_arn.

Point-in-Time Recovery (PITR)

Continuous backups enabled by default (enable_point_in_time_recovery = true) for up to 35 days of recovery.

DAX IAM Least Privilege

DAX cluster IAM role is scoped to only the specific table ARN and its indexes, preventing cross-table access.

DAX Server-Side Encryption

DAX cluster has server-side encryption enabled by default for cache data protection.

Industry Adaptation

Healthcare

PITR and KMS encryption for PHI compliance. TTL for automatic expiration of temporary session data. Global tables for multi-region disaster recovery.

Finance

DynamoDB Streams for real-time transaction event processing. DAX caching for high-frequency trading lookups. Auto scaling for market-hours traffic spikes.

Government

CMK encryption for data sovereignty. PITR for audit trail preservation. Contributor Insights for access pattern monitoring and compliance reporting.

Retail

On-demand billing for seasonal traffic. DAX for sub-millisecond product catalog reads. GSIs for flexible querying by category, price, or availability.

SaaS

Multi-tenant table design with partition key isolation. TTL for tenant data lifecycle management. Kinesis streaming for real-time usage analytics.

Production Readiness Checklist

  • Enable Point-in-Time Recovery for all production tables
  • Use customer-managed KMS keys for sensitive data encryption
  • Configure auto scaling for provisioned-mode tables to handle traffic spikes
  • Enable Contributor Insights to identify hot keys and throttling patterns
  • Set up DynamoDB Streams or Kinesis for change data capture pipelines
  • Configure global table replicas for multi-region availability requirements
  • Enable DAX for read-heavy workloads requiring microsecond latency
  • Set appropriate TTL attributes for data that should auto-expire
  • Design GSI/LSI access patterns before table creation (LSIs are immutable)
  • Apply consistent tagging for cost allocation across billing modes
